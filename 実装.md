# 実装進捗

## ビルド方法

```bash
# 事前準備: VCPKG_ROOT 環境変数を設定
# テスト用フォントのダウンロード
make fontdata

# CMake 構成（初回または CMakeLists.txt 変更時）
make prebuild                      # OS 自動検出
make prebuild PRESET=x64-windows   # プリセット明示指定

# ビルド
make build
make build BUILD_TYPE=Debug        # デバッグビルド

# サンプル実行（プロジェクトルートから）
./build/x64-windows/Release/sample_render.exe
# → output.bmp が生成される
```

## 既知の課題

### フェイク斜体 — 実装済み

`GlyphRenderer` が `FontFakery::isFakeItalic()` を参照し、`renderPath` でグリフパスにシアー変換（skewX = -0.25、Android と同じ約14度）を適用する。パスキャッシュはシアー前の形状を保持し、描画時に変換するため追加のキャッシュ消費はない。

### フェイク太字 — 実装済み

`GlyphRenderer` が `FontFakery::isFakeBold()` を参照し、グリフパスに同色ストローク（フォントサイズの 1/24 幅）を重ねることで太字をシミュレートする。minikin は要求ウェイト ≥ 600 かつ実際のウェイトとの差 ≥ 200 の場合に自動的にフェイクボールドフラグを立てる。

### バリアブルフォント — 実装済み

`FontFace` が FreeType の `FT_Get_MM_Var` でバリアブルフォントの軸情報を取得し、`setVariations()` で `FT_Set_Var_Design_Coordinates` を呼び出してウェイト等の軸値を設定する。`FontManager::registerVariableFont()` で同じフォントファイルから異なるウェイトの FontFace を複数作成し、minikin の FontFamily に登録することで、テキストスタイルの `fontWeight` 指定に応じて適切なウェイトが自動選択される。

対応優先度:
1. バリアブルフォントの wght 軸で正確なウェイトを描画（最高品質）
2. ウェイト別の固定フォントファイルが登録されていれば minikin が自動選択
3. 上記いずれもない場合はフェイクボールドでストロークシミュレーション
