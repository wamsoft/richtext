# 実装進捗

## ビルド方法

```bash
# 事前準備: VCPKG_ROOT 環境変数を設定
# テスト用フォントのダウンロード
make fontdata

# CMake 構成（初回または CMakeLists.txt 変更時）
make prebuild                      # OS 自動検出
make prebuild PRESET=x64-windows   # プリセット明示指定

# ビルド
make build
make build BUILD_TYPE=Debug        # デバッグビルド

# サンプル実行（プロジェクトルートから）
./build/x64-windows/Release/sample_render.exe
# → output.bmp が生成される
```

### フォントローダー — 実装済み

ライブラリはファイルシステムに直接アクセスせず、利用側がローダー関数を登録する。

- `setFontDataLoader()` — バッファ方式。`shared_ptr<vector<uint8_t>>` を返す関数を登録。**優先使用**
- `setFontStreamLoader()` — ストリーム方式。`FT_Stream` を返す関数を登録。バッファローダーのフォールバック

`registerFont(fileName, name)` / `registerVariableFont(fileName, name, weight)` でフォント登録時にローダーが呼び出される。`fileName` はローダーに渡される識別子（ファイル名やパス）、`name` はライブラリ内部の登録名。

サンプルでは `std::ifstream` でファイル読み込みするバッファローダーを登録し、`./data/` パスの解決はローダー内で行う。

## 既知の課題

### フェイク斜体 — 実装済み

`GlyphRenderer` が `FontFakery::isFakeItalic()` を参照し、`renderPath` でグリフパスにシアー変換（skewX = -0.25、Android と同じ約14度）を適用する。パスキャッシュはシアー前の形状を保持し、描画時に変換するため追加のキャッシュ消費はない。

### フェイク太字 — 実装済み

`GlyphRenderer` が `FontFakery::isFakeBold()` を参照し、グリフパスに同色ストローク（フォントサイズの 1/24 幅）を重ねることで太字をシミュレートする。minikin は要求ウェイト ≥ 600 かつ実際のウェイトとの差 ≥ 200 の場合に自動的にフェイクボールドフラグを立てる。

### バリアブルフォント — 実装済み

`FontFace` が FreeType の `FT_Get_MM_Var` でバリアブルフォントの軸情報を取得し、`setVariations()` で `FT_Set_Var_Design_Coordinates` を呼び出して軸値を設定する。`FontManager::registerVariableFont()` で同じフォントファイルから異なるウェイト・スタイルの FontFace を複数作成し、minikin の FontFamily に登録。

**対応軸:**

- **wght（ウェイト）**: `fontWeight` 指定で minikin が自動選択。未対応時はフェイクボールド
- **ital（イタリック）**: `italic=true` + `registerVariableFont(path, name, w, true)` で Italic FontFace を登録。未対応時はフェイクイタリック（skewX=-0.25）
- **wdth（字幅）**: `fontWidth` 指定。`GetHorizontalAdvance` 内で wdth 軸を設定し、軸でカバーできない分は fakeScaleX で補う。描画時も同様に `GlyphRenderer` が wdth 軸 + fakeScaleX を適用

**minikin 修正:**

- `MinikinPaint` に `fontWidth` フィールドを追加（`LayoutCacheKey` にも反映）
- `FontFace::GetHorizontalAdvance` で `paint.fontWidth` を参照し、wdth 軸設定 + fake スケールを適用

**タグ対応:**

- `<font width="75">` で fontWidth を指定可能
