# RichText ライブラリ設計資料

## 1. 概要

吉里吉里用のリッチテキストレンダリングライブラリ。minikinによるテキストレイアウト、FreeTypeによるグリフパス化、thorvgによるベクター描画を組み合わせて、高品質なテキスト描画を実現する。

### 1.1 主要機能

- **1行テキストレンダリング**: シンプルな1行テキストの高速描画
- **パラグラフレンダリング**: 矩形領域内での複数行テキスト描画（上下左右アライン対応）
- **多言語対応**: 複数言語の混在、RTL（右から左）テキスト対応
- **フォントフォールバック**: 複数フォントを束ねて文字カバレッジを補完
- **絵文字対応**: カラー絵文字を含むテキスト描画
- **スタイル指定**: フォント、サイズ、色、スペーシングなどのスタイル制御
- **装飾描画**: ストローク（縁取り）、塗りを複数回重ねる装飾処理
- **グリフキャッシュ**: グリフ単位でのビットマップキャッシュによる高速化

## 2. 使用ライブラリと役割

| ライブラリ | 役割 |
|-----------|------|
| **minikin** | テキストレイアウトエンジン（行分割、グリフ配置座標計算） |
| **FreeType** | フォント読み込み、グリフのアウトラインパス取得 |
| **thorvg** | ベクターグラフィックス描画（パスの塗り・ストローク） |
| **harfbuzz** | 複雑なテキスト整形（minikin内部で使用） |
| **ICU** | Unicode処理、行分割ルール（minikin内部で使用） |

### 2.1 データフロー

```
テキスト入力 (UTF-16)
        ↓
[minikin] テキストレイアウト
  - フォント選択（フォールバック処理）
  - 行分割計算
  - グリフID・配置座標の算出
        ↓
[FreeType] グリフパス化
  - グリフIDからアウトラインパス取得
  - カラー絵文字はビットマップ取得
        ↓
[thorvg] ベクター描画
  - パスにストローク/塗りを適用
  - ビットマップへのレンダリング
        ↓
出力ビットマップ (ARGB)
```

### 5.2 インラインタグ仕様

テキスト中にHTMLライクなタグを埋め込むことで、部分的にスタイルを変更できます。

#### 5.2.1 対応タグ一覧

| タグ | 説明 | 属性 |
|-----|------|------|
| `<font>` | フォント変更 | `face`, `size`, `weight` |
| `<color>` | 色変更 | `value` (ARGB) または `r`,`g`,`b`,`a` |
| `<b>` | 太字 | - |
| `<i>` | 斜体 | - |
| `<u>` | 下線 | - |
| `<s>` | 取り消し線 | - |
| `<ruby>` | ルビ（振り仮名） | `text` |
| `<sup>` | 上付き文字 | - |
| `<sub>` | 下付き文字 | - |
| `<style>` | 登録済みスタイル参照 | `name` |
| `<br>` | 改行 | - |
| `<sp>` | スペース | `width` (em単位) |
| `<outline>` | 縁取り | `color`, `width` |
| `<shadow>` | 影 | `color`, `x`, `y` |

#### 5.2.2 タグ詳細

**`<font>` タグ**
```
<font face='フォント名' size=24 weight=700>テキスト</font>
```
- `face`: フォントコレクション名または登録済みフォント名
- `size`: フォントサイズ（ピクセル）
- `weight`: フォントウェイト（100-900）

**`<color>` タグ**
```
<color value=0xFFFF0000>赤いテキスト</color>
<color value='#FF0000'>赤いテキスト</color>
<color r=255 g=0 b=0>赤いテキスト</color>
```
- `value`: ARGB値（0xAARRGGBB形式）または#RRGGBB形式
- `r`, `g`, `b`, `a`: 個別の色成分（0-255）

**`<ruby>` タグ**
```
<ruby text='とうきょう'>東京</ruby>
```
- `text`: ルビテキスト
- ルビのフォントサイズはデフォルトで親文字の50%

**`<style>` タグ**
```
<style name='emphasis'>強調テキスト</style>
```
- `name`: 事前に登録したスタイル名を参照
- TextStyle と Appearance の両方が適用される

**`<outline>` タグ**
```
<outline color=0xFF000000 width=2>縁取り付きテキスト</outline>
```
- `color`: 縁取りの色（ARGB）
- `width`: 縁取りの幅（ピクセル）

**`<shadow>` タグ**
```
<shadow color=0x80000000 x=2 y=2>影付きテキスト</shadow>
```
- `color`: 影の色（ARGB）
- `x`, `y`: 影のオフセット（ピクセル）

#### 5.2.3 タグのネスト

タグはネスト可能です：
```
<color value=0xFFFF0000><b>赤くて太い</b></color>
```

#### 5.2.4 エスケープ

特殊文字のエスケープ：
- `&lt;` → `<`
- `&gt;` → `>`
- `&amp;` → `&`
- `&quot;` → `"`

### 5.3 タグ付きテキスト文法仕様

#### 5.3.1 EBNF 文法定義

```ebnf
tagged_text     = { text_element } ;
text_element    = plain_text | tag_element | escape_sequence ;

plain_text      = { character - ('<' | '&') } ;
escape_sequence = '&lt;' | '&gt;' | '&amp;' | '&quot;' ;

tag_element     = open_tag , tagged_text , close_tag
                | self_closing_tag ;

open_tag        = '<' , tag_name , { attribute } , '>' ;
close_tag       = '</' , tag_name , '>' ;
self_closing_tag = '<' , tag_name , { attribute } , '/>' ;

tag_name        = 'font' | 'color' | 'b' | 'i' | 'u' | 's' 
                | 'ruby' | 'sup' | 'sub' | 'style' | 'br' | 'sp'
                | 'outline' | 'shadow' ;

attribute       = attr_name , '=' , attr_value ;
attr_name       = identifier ;
attr_value      = number | quoted_string | hex_number ;

number          = digit , { digit } , [ '.' , digit , { digit } ] ;
hex_number      = '0x' , hex_digit , { hex_digit } ;
quoted_string   = ("'" , { character - "'" } , "'")
                | ('"' , { character - '"' } , '"') ;
identifier      = letter , { letter | digit | '_' } ;
```

#### 5.3.2 属性値の型

| 属性値の形式 | 例 | 説明 |
|-------------|-----|------|
| 整数 | `24`, `700` | 10進数整数 |
| 小数 | `1.5`, `0.8` | 浮動小数点数 |
| 16進数 | `0xFFFF0000` | ARGB色値 |
| 文字列 | `'NotoSansCJK'`, `"text"` | シングルまたはダブルクォート |
| カラーコード | `'#FF0000'` | #RRGGBB形式 |

### 5.4 タグパーサー詳細設計

#### 5.4.1 TagParser クラス

```cpp
class TagParser {
public:
    // テキストスパン（スタイル適用区間）
    struct TextSpan {
        size_t start;               // 開始位置（プレーンテキスト内）
        size_t end;                 // 終了位置
        TextStyle style;            // テキストスタイル
        Appearance appearance;      // 描画外観
        
        // 特殊効果
        bool hasRuby = false;
        std::u16string rubyText;
        
        bool hasUnderline = false;
        bool hasStrikethrough = false;
    };
    
    // パース結果
    struct ParseResult {
        std::u16string plainText;           // タグ除去後のプレーンテキスト
        std::vector<TextSpan> spans;        // スタイル適用区間
        std::vector<ParagraphLayout::StyleRun> styleRuns;  // minikin用StyleRun
    };
    
    // パースオプション
    struct ParseOptions {
        bool allowNestedTags = true;        // ネストタグを許可
        bool ignoreUnknownTags = true;      // 未知のタグを無視
        float rubyScale = 0.5f;             // ルビのサイズ倍率
        float supScale = 0.7f;              // 上付き文字の倍率
        float subScale = 0.7f;              // 下付き文字の倍率
    };
    
    TagParser();
    
    // パースオプションの設定
    void setOptions(const ParseOptions& options);
    
    // タグ付きテキストの解析
    ParseResult parse(
        const std::u16string& taggedText,
        const TextStyle& defaultStyle,
        const Appearance& defaultAppearance,
        const std::map<std::string, TextStyle>& namedStyles = {},
        const std::map<std::string, Appearance>& namedAppearances = {}
    );
    
    // スタイル継承を考慮したStyleRun配列の生成
    std::vector<ParagraphLayout::StyleRun> buildStyleRuns(
        const ParseResult& result
    );

private:
    ParseOptions options_;
    
    // 内部パース状態
    struct ParseState {
        std::u16string plainText;
        std::vector<TextSpan> spans;
        
        // スタイルスタック（ネスト管理）
        struct StyleStackEntry {
            std::string tagName;
            TextStyle style;
            Appearance appearance;
            size_t startPos;
        };
        std::vector<StyleStackEntry> styleStack;
    };
    
    // タグの解析
    bool parseTag(const std::u16string& text, size_t& pos, ParseState& state,
                  const TextStyle& defaultStyle,
                  const Appearance& defaultAppearance,
                  const std::map<std::string, TextStyle>& namedStyles,
                  const std::map<std::string, Appearance>& namedAppearances);
    
    // 属性の解析
    std::map<std::string, std::string> parseAttributes(
        const std::u16string& text, size_t& pos);
    
    // タグからスタイル変更を適用
    TextStyle applyFontTag(const TextStyle& current,
                           const std::map<std::string, std::string>& attrs);
    Appearance applyColorTag(const Appearance& current,
                             const std::map<std::string, std::string>& attrs);
    Appearance applyOutlineTag(const Appearance& current,
                               const std::map<std::string, std::string>& attrs);
    Appearance applyShadowTag(const Appearance& current,
                              const std::map<std::string, std::string>& attrs);
};
```

#### 5.4.2 スタイル継承の仕組み

タグがネストされた場合、内側のタグは外側のタグのスタイルを継承します。

```
デフォルトスタイル: fontSize=16, color=白
    │
    └─ <font size=24>
           │  → fontSize=24, color=白（継承）
           │
           └─ <color value=0xFFFF0000>
                  │  → fontSize=24（継承）, color=赤
                  │
                  └─ <b>
                         → fontSize=24（継承）, color=赤（継承）, weight=700
```

#### 5.4.3 スタイル適用の優先順位

1. デフォルトスタイル（ベース）
2. `<style name='...'>` で参照した名前付きスタイル
3. 個別タグによる上書き（`<font>`, `<color>` など）
4. 内側のタグが外側より優先

#### 5.4.4 TextSpan から StyleRun への変換

```cpp
// ParseResult からレンダリング用の StyleRun 配列を生成
std::vector<ParagraphLayout::StyleRun> TagParser::buildStyleRuns(
    const ParseResult& result
) {
    std::vector<ParagraphLayout::StyleRun> runs;
    
    // スパンをソートしてマージ
    // 同じスタイルの連続スパンは1つにまとめる
    for (const auto& span : result.spans) {
        ParagraphLayout::StyleRun run;
        run.start = span.start;
        run.end = span.end;
        run.style = span.style;
        runs.push_back(run);
    }
    
    return runs;
}
```

### 5.5 使用例：タグからスタイル定義を生成

#### 5.5.1 基本的な使用フロー

```cpp
// 1. TagParser を作成
TagParser parser;

// 2. デフォルトスタイルを準備
TextStyle defaultStyle;
defaultStyle.fontCollection = fontCollection;
defaultStyle.fontSize = 16.0f;

Appearance defaultAppearance;
defaultAppearance.addFill(0xFFFFFFFF);  // 白色

// 3. 名前付きスタイルを準備（オプション）
std::map<std::string, TextStyle> namedStyles;
std::map<std::string, Appearance> namedAppearances;

TextStyle emphasisStyle = defaultStyle;
emphasisStyle.fontWeight = 700;
namedStyles["emphasis"] = emphasisStyle;

Appearance emphasisAppearance;
emphasisAppearance.addStroke(0xFF000000, 2);
emphasisAppearance.addFill(0xFFFFFF00);
namedAppearances["emphasis"] = emphasisAppearance;

// 4. タグ付きテキストをパース
std::u16string taggedText = u"これは<font size=24>大きな</font>文字と"
                            u"<color value=0xFFFF0000>赤い</color>文字と"
                            u"<style name='emphasis'>強調</style>です。";

TagParser::ParseResult result = parser.parse(
    taggedText,
    defaultStyle,
    defaultAppearance,
    namedStyles,
    namedAppearances
);

// 5. 結果を使用
// result.plainText = "これは大きな文字と赤い文字と強調です。"
// result.spans には各区間のスタイル情報が格納

// 6. レンダリング
textRenderer.drawParagraph(
    result.plainText,
    rect,
    ParagraphLayout::HAlign::Left,
    ParagraphLayout::VAlign::Top,
    result.styleRuns,
    defaultAppearance
);
```
