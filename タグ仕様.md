# タグパーサー仕様

`TagParser` はHTMLライクなタグ付きテキストを解析し、テキストスタイルと描画外観の区間（スパン）に分解する。

---

| タグ名 | エイリアス | 種別 | 説明 |
|---|---|---|---|
| `font` | — | 通常タグ | フォント変更 |
| `b` | `strong` | 通常タグ | 太字（weight=700） |
| `i` | `em` | 通常タグ | 斜体 |
| `u` | — | 通常タグ | 下線 |
| `s` | `strike`, `del` | 通常タグ | 取り消し線 |

| `color` | — | 通常タグ | テキスト色変更 |
| `outline` | — | 通常タグ | 縁取り（ストローク追加） |
| `shadow` | — | 通常タグ | 影（オフセット付き塗り追加） |
| `ruby` | — | 通常タグ | ルビ（振り仮名） |
| `style` | — | 通常タグ | 名前付きスタイル参照 |

## 各タグの属性

### `<font>`

フォント・テキストスタイルの変更。

| 属性 | 型 | 説明 |
|---|---|---|
| `face` | string | フォントファミリー名。`FontManager` の名前付きコレクション（`registerCollection` で登録）を優先参照し、見つからなければ単一フォント名として `createCollection` でコレクションを作成する |
| `size` | float | フォントサイズ（ピクセル） |
| `weight` | int | フォントウェイト（100〜900） |
| `spacing` | float | 字間（letterSpacing） |
| `width` | float | フォント幅 |

使用例:
```
<font face='serif' size=24>明朝体のテキスト</font>
<font weight=700>太字</font>
```

### `<b>` / `<strong>`

太字。`fontWeight` を 700 に設定する。属性なし。

### `<i>` / `<em>`

斜体。`italic` を true に設定する。属性なし。

### `<u>`

下線。属性なし。

### `<s>` / `<strike>` / `<del>`

取り消し線。属性なし。

### `<sup>`

上付き文字。フォントサイズを `supScale` 倍（デフォルト 0.7）にし、Y方向に `supOffset`（デフォルト -0.4em）オフセットする。属性なし。

### `<sub>`

下付き文字。フォントサイズを `subScale` 倍（デフォルト 0.7）にし、Y方向に `subOffset`（デフォルト 0.2em）オフセットする。属性なし。

### `<color>`

テキスト色の変更。2通りの指定方法がある。

**方法A — `value` 属性:**

| 属性 | 型 | 説明 |
|---|---|---|
| `value` | color | 色値 |

**方法B — 個別 RGBA:**

| 属性 | 型 | デフォルト | 説明 |
|---|---|---|---|
| `r` | int | 0 | 赤（0〜255） |
| `g` | int | 0 | 緑（0〜255） |
| `b` | int | 0 | 青（0〜255） |
| `a` | int | 255 | アルファ（0〜255） |

`value` 属性が存在する場合、`r`/`g`/`b`/`a` は無視される。

**共通属性:**

| 属性 | 型 | デフォルト | 説明 |
|---|---|---|---|
| `x` | float | 0.0 | Xオフセット（`add` 時のみ有効） |
| `y` | float | 0.0 | Yオフセット（`add` 時のみ有効） |
| `add` | flag | — | 存在すると追加モード（値不要） |

- **通常（`add` なし）**: `Appearance::setColor()` — 既存の通常 Fill をすべて置換して新しい色を設定する。
- **追加モード（`add` あり）**: `Appearance::addColor()` — 既存の色を残したまま、最前面に色レイヤーを追加する。`x`/`y` でオフセット指定も可能。

使用例:
```
<color value=0xFFCC0000>赤色</color>
<color r=0 g=128 b=255>青色</color>
<color value=0xFFFFDD00>黄色<color value=0x880000FF x=2 y=2 add>半透明青を重ねる</color></color>
```

### `<outline>`

縁取り（ストローク）の追加。

| 属性 | 型 | デフォルト | 説明 |
|---|---|---|---|
| `color` | color | `0xFF000000`（黒） | 縁取り色 |
| `width` | float | 2.0 | 縁取り幅（ピクセル） |
| `x` | float | 0.0 | Xオフセット |
| `y` | float | 0.0 | Yオフセット |
| `add` | flag | — | 存在すると追加モード（値不要） |

線結合は `Round` 固定。

- **通常（`add` なし）**: `Appearance::setOutline()` — 既存の Stroke をすべて置換して新しい縁取りを設定する。
- **追加モード（`add` あり）**: `Appearance::addOutline()` — 既存の Stroke を残したまま、最背面に縁取りレイヤーを追加する。外側の太い縁取りを先に `add` し、内側の細い縁取りを後から `add` することで二段縁取りを実現する。

使用例:
```
<outline color=0xFFFFFFFF width=3>白縁取り</outline>
<outline color=0xFF000000 width=6><outline color=0xFFFFFFFF width=2 add>二段縁取り</outline></outline>
```

### `<shadow>`

影の追加。

| 属性 | 型 | デフォルト | 説明 |
|---|---|---|---|
| `color` | color | `0x80000000`（半透明黒） | 影の色 |
| `x` | float | 2.0 | Xオフセット |
| `y` | float | 2.0 | Yオフセット |
| `add` | flag | — | 存在すると追加モード（値不要） |

- **通常（`add` なし）**: `Appearance::setShadow()` — 既存の影 Fill をすべて置換して新しい影を設定する。
- **追加モード（`add` あり）**: `Appearance::addShadow()` — 既存の影を残したまま、最背面に影レイヤーを追加する。

使用例:
```
<shadow color=0x80000000 x=3 y=3>影付きテキスト</shadow>
<shadow color=0x40000000 x=6 y=6><shadow color=0x80000000 x=2 y=2 add>二段影</shadow></shadow>
```

### `<ruby>`

ルビ（振り仮名）の付与。

| 属性 | 型 | 説明 |
|---|---|---|
| `text` | string | ルビテキスト |

使用例:
```
<ruby text='かんじ'>漢字</ruby>
```

### `<style>`

名前付きスタイル・外観の参照。`parse()` に渡した `namedStyles` / `namedAppearances` マップから検索する。

| 属性 | 型 | 説明 |
|---|---|---|
| `name` | string | スタイル名 |

使用例:
```
<style name='highlight'>重要なテキスト</style>
```

### `<br>`

改行（`\n`）を挿入する。Void タグのため閉じタグ不要。

### `<sp>`

スペースを挿入する。Void タグのため閉じタグ不要。

| 属性 | 型 | デフォルト | 説明 |
|---|---|---|---|
| `width` | int | 1 | スペース数（最小 1） |

### `<delay>`

後続文字の表示遅延を変更する。Void タグのため閉じタグ不要。

| 属性 | 型 | 説明 |
|---|---|---|
| `value` | string | 遅延値。末尾に `%` があればパーセント指定（標準時間に対する割合）、なければ ms 指定 |

使用例:
```
通常速度<delay value='50%'/>半分の速度<delay value='200'/>200ms固定
```

### `<wait>`

文字なしの時間待ちを挿入する。Void タグのため閉じタグ不要。

| 属性 | 型 | 説明 |
|---|---|---|
| `value` | string | 待ち時間。末尾に `%` があればパーセント指定（1文字時間に対する割合）、なければ ms 指定 |

使用例:
```
テキスト<wait value='500'/>500ms待ち
テキスト<wait value='200%'/>2文字分待ち
```

### `<sync>`

絶対時間またはラベルによる同期ポイントを挿入する。Void タグのため閉じタグ不要。

| 属性 | 型 | 説明 |
|---|---|---|
| `value` | string | 全部数値なら ms（絶対同期時間）、それ以外ならラベル名（`LabelResolver` コールバックで時間に解決される） |

使用例:
```
テキスト<sync value='3000'/>3秒時点から再開
テキスト<sync value='chapter2'/>ラベルで同期
```

### `<keywait>`

キー入力待ちポイントを挿入する。属性なし。Void タグのため閉じタグ不要。

使用例:
```
ここで待ち<keywait/>次のテキスト
```

### `<link>`

リンク領域を定義する。通常タグ（閉じタグ必要）。

| 属性 | 型 | 説明 |
|---|---|---|
| `name` | string | リンク名（識別子） |

リンク範囲は `ParseResult::links`（`LinkInfo` 配列）に記録され、`StyledLayout::buildLinkRegions()` で矩形領域に変換できる。

使用例:
```
<link name='url1'>クリックできるテキスト</link>
```

### `<graph>`

グラフィック文字（インライン画像）を挿入する。Void タグのため閉じタグ不要。プレーンテキスト内に U+FFFC（Object Replacement Character）が挿入される。

| 属性 | 型 | 説明 |
|---|---|---|
| `name` | string | 画像名 |
| `width` | float | 幅（ピクセル） |
| `height` | float | 高さ（ピクセル） |

グラフィック情報は `ParseResult::graphics`（`GraphInfo` 配列）に記録される。

使用例:
```
テキスト<graph name='icon.png' width='24' height='24'/>画像付き
```

### `<eval>`

変数評価タグ。`EvalCallback` コールバックを呼び出し、結果の文字列をプレーンテキストに挿入する。Void タグのため閉じタグ不要。

| 属性 | 型 | 説明 |
|---|---|---|
| `name` | string | 評価する変数名（コールバックに渡される） |
| `alt` | string | 代替表示文字列（コールバック未登録または空文字を返した場合に使用） |

表示優先順位:
1. `EvalCallback` が登録されていて非空文字列を返した場合 → その文字列
2. `alt` 属性が指定されている場合 → `alt` の値
3. いずれもない場合 → `name` の値をそのまま表示

`EvalCallback` は `TagParser::setEvalCallback()` または `StyledLayout::setEvalCallback()` で設定する。

使用例:
```
<eval name='playerName' alt='名無し'/>さん、こんにちは
現在のスコア: <eval name='score'/>
```

## 色の指定形式

`color` 型の属性は以下の形式を受け付ける:

| 形式 | 例 | 説明 |
|---|---|---|
| `0xRRGGBB` | `0xCC0000` | 6桁16進（アルファ=255 補完） |
| `0xAARRGGBB` | `0x80CC0000` | 8桁16進 |
| `#RRGGBB` | `#CC0000` | 6桁16進（アルファ=255 補完） |
| `#AARRGGBB` | `#80CC0000` | 8桁16進 |
| `RRGGBB` | `CC0000` | プレフィックスなし |

パース失敗時は `0xFFFFFFFF`（白）にフォールバックする。

## 自己閉じタグ

任意のタグで `<tag ... />` と書くと自己閉じタグとして扱われ、テキスト区間は生成されない（スタックに積まれない）。

---

## ネスト・スタイル継承

- タグはスタック構造で管理される。開きタグで push、閉じタグで pop
- 子タグは親タグのスタイルを継承し、差分のみ変更する
- 閉じタグ時、スタックが空なら `defaultStyle` / `defaultAppearance` に復元
- タグの不一致は寛容モード（デフォルト）ではスタックトップを閉じる
- 未閉じタグはパース終了時にスパンとして確定される
- タグで覆われない領域にはデフォルトスタイルのスパンが自動挿入される

---

## 描画順序

`Appearance` 内の `DrawStyle` は以下の順で描画される（後のものが上に描画）:

1. **影 Fill**（オフセット付き Fill）— 最背面
2. **Stroke**（縁取り）
3. **通常 Fill**（オフセットなし）— 最前面

この順序は `Appearance` の `set*` / `add*` メソッドが自動的に管理する。

### Appearance レイヤー操作メソッド

「一番下のオフセット 0 Fill」（`findBottomMainFillIndex()`）がレイヤー境界となり、それより下が装飾（影・縁取り）、それ以上がテキスト色である。

| メソッド | 動作 |
|---|---|
| `setColor(argb)` | 境界以上の Fill をすべて削除し、新しい色を最前面に追加 |
| `addColor(argb, ox, oy)` | 最前面に色 Fill を追加（既存はそのまま） |
| `setOutline(argb, w, ox, oy)` | 境界より下の Stroke をすべて削除し、新しい縁取りを追加 |
| `addOutline(argb, w, ox, oy)` | 既存 Stroke の最背面に縁取りを追加（外側から太い順に重ねる用途） |
| `setShadow(argb, ox, oy)` | 境界より下の Fill をすべて削除し、新しい影を追加 |
| `addShadow(argb, ox, oy)` | 最背面（インデックス 0）に影を追加（既存はそのまま） |

## フォントファミリー切り替え

`<font face='...'>` でフォントファミリーを切り替えられる。

### 名前付きコレクション（推奨）

`FontManager::registerCollection()` で事前にフォールバックチェーン付きのコレクションを登録し、`face` 属性でその名前を指定する。

```cpp
// 事前登録
fm.registerCollection("sans",  {"ja", "sans", "emoji"});
fm.registerCollection("serif", {"serif-ja", "serif", "emoji"});
```

```
ゴシック体と<font face='serif'>明朝体</font>の切り替え
```

名前付きコレクションが見つからない場合は、単一フォント名として `createCollection({name})` で簡易コレクションが作成される（フォールバックなし）。

## ParseOptions

| フィールド | 型 | デフォルト | 説明 |

## EvalCallback

`eval` タグ用のコールバック型。

```cpp
using EvalCallback = std::function<std::u16string(const std::u16string& name)>;
```

`TagParser::setEvalCallback()` または `StyledLayout::setEvalCallback()` で `layout()` / `parse()` 前に設定する。
